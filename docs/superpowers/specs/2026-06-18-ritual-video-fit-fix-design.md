# 求签视频完整显示修复设计

日期：2026-06-18
状态：方案 A 已确认，待规格审核

## 根因

求签视频原始尺寸为 720×1280（9:16），桌面视口实测为 1280×720（16:9）。当前视频使用 `object-cover` 并放大至 103%，横屏下实际只能看到约 32% 的视频高度，导致签筒顶部和底部被裁切。

## 修复方案

采用双层视频构图：

1. 背景视频继续覆盖整个视口，增加模糊、降饱和和透明度，仅承担动态氛围。
2. 前景视频使用 `object-contain` 完整等比显示，不缩放裁切。
3. 前景媒体区为底部签文和进度条预留安全空间，避免完整签筒被文字遮挡。
4. 水墨远山、雾气、暗角和印章继续位于视觉叠层，不改变当前仪式风格。
5. 视频加载失败时，前景与背景都退化为本地签筒海报，求签流程不受影响。

## 响应式布局

- 桌面横屏：前景视频置中，最大高度扣除底部文案区；左右空间由模糊背景和水墨层填充。
- 移动竖屏：前景视频仍使用 `contain`，在安全区内尽量放大，但完整画面优先于铺满。
- 前景视频不得使用 `scale`、`object-cover` 或负边距制造裁切。
- 页面不得产生水平滚动条。

## 组件边界

只修改 `DivinationRitualOverlay` 的媒体布局，不修改 `App.tsx` 中的求签状态、计时器、接口请求、结果弹窗或历史记录逻辑。

## 验收标准

1. 1280×720 桌面视口中，前景视频完整显示 720×1280 的全部画面。
2. 390×844 移动视口中，前景视频完整显示且不超出视口。
3. 底部签文和进度条不遮挡签筒主体。
4. 背景保持全屏动态水墨效果，不出现单调黑边。
5. 页面中仍只存在必要的视频实例，不重复播放音轨；全部视频静音且行内播放。
6. 视频失败时仍显示海报、水墨层、提示文案和进度。
7. 自动测试、TypeScript 检查、生产构建和浏览器验证通过。

## 非目标

- 不更换视频文件。
- 不修改求签时长和文案轮播逻辑。
- 不处理浏览器或 Codex 外挂宠物等非应用 DOM 元素。
